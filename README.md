# Zynq AXI Peripherals

Bare-metal embedded C++ applications on the **Xilinx Zynq-7000 ZC702** board, a SoC containing ARM Cortex-A9 PS and FPGA PL, demonstrating hardware/software co-design across the processor–programmable logic (PS/PL) boundary using the AXI4 protocol. Each module is self-contained and targets a distinct embedded systems concept: memory-mapped I/O, hardware timers, interrupt-driven programming, and DMA-accelerated memory transfers.


## Platform

| | |
| **Board** | Xilinx ZC702 Evaluation Board |
| **SoC** | Zynq-7000 XC7Z020 (ARM Cortex-A9 Processor + Artix-7 FPGA) |
| **Toolchain** | Xilinx ISE 14.7 / XPS / SDK |
| **Language** | C++ (bare-metal), VHDL |
| **Interface** | AXI4 / AXI4-Lite protocol |


## Modules

### [`axi-gpio-pl`](./axi-gpio-pl) — GPIO with Programmable Logic
Implements an 8-bit AND gate in VHDL on the FPGA fabric and drives it from the ARM processor via a 16-bit AXI GPIO port. Demonstrates PS/PL integration: VHDL logic synthesis, UCF pin constraints, and memory-mapped GPIO access from C++.

**Key concepts:** PS/PL co-design, AXI GPIO, VHDL instantiation, UCF constraints


### [`axi-timer`](./axi-timer) — AXI Hardware Timer
Direct register-level programming of the AXI Timer IP core in two modes:

- **Capture mode** — measures clock cycles elapsed between two external hardware events (push button SW5), demonstrating hardware event timestamping
- **PWM mode** — generates a user-configurable PWM signal to control LED brightness, with interactive period and duty cycle input

**Key concepts:** Direct register access via volatile pointers, hardware timers, PWM generation, capture mode, AXI register map


### [`axi-interrupt`](./axi-interrupt) — ARM Interrupt Handling via GIC
Configures the ARM Generic Interrupt Controller (GIC / XScuGic) to handle periodic interrupts generated by the AXI Timer. Binds a user-defined C++ ISR to the timer interrupt line (IRQ_F2P), demonstrating interrupt-driven programming on a Cortex-A9.

**Key concepts:** ARM GIC (XScuGic), ISR registration, interrupt-driven execution, FPGA-to-processor interrupt routing


### [`axi-cdma`](./axi-cdma) — DMA-Accelerated Memory Transfer 
Configures the AXI Central DMA (CDMA) IP core to perform high-throughput memory-to-memory transfers via two AXI high-performance slave ports (HP0 / HP2). An AXI Timer benchmarks the transfer against a software (CPU) loop copy across multiple array sizes.

**Key concepts:** AXI CDMA, DMA vs CPU memory transfer, AXI HP ports, cache coherency (`Xil_DCacheFlush`, `Xil_DCacheInvalidateRange`), performance benchmarking

#### Benchmark Results (50 MHz clock, Zynq ZC702)

| Array Size (u32) | SW Cycles | CDMA Cycles | Speedup |
|-----------------|-----------|-------------|---------|
| 1,024           | 1,513     | 1,120       | 1.35×   |
| 4,096           | 6,016     | 2,646       | 2.27×   |
| 8,192           | 12,315    | 4,704       | 2.62×   |
| 16,384          | 24,762    | 8,792       | 2.82×   |
| 32,767          | 49,809    | 16,982      | 2.93×   |
| 1,048,576       | 1,623,301 | 524,888     | 3.09×   |
| 2,097,151       | 3,243,081 | 1,049,172   | 3.09×   |

> CDMA achieves up to **3.09× speedup** over CPU-based copy, with speedup increasing with transfer size as DMA overhead becomes negligible relative to transfer time.
