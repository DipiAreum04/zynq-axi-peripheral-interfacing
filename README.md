# Zynq AXI Peripherals

Bare-metal embedded C++ applications on the **Xilinx Zynq-7000 ZC702** board, a SoC containing ARM Cortex-A9 PS and FPGA PL, demonstrating hardware/software co-design across the processor–programmable logic (PS/PL) boundary using the AXI4 protocol. Each module is self-contained and targets a distinct embedded systems concept: memory-mapped I/O, hardware timers, interrupt-driven programming, and DMA-accelerated memory transfers. Developed as part of the Microprocessor-based Systems course at Concordia University.

## Modules

### [`axi-gpio-pl`](./axi-gpio-pl) — GPIO with Programmable Logic
Implements an 8-bit AND gate in VHDL on the FPGA fabric and drives it from the ARM processor via a 16-bit AXI GPIO port. Demonstrates PS/PL integration: VHDL logic synthesis, UCF pin constraints, and memory-mapped GPIO access from C++.

**Key concepts:** PS/PL co-design, AXI GPIO, VHDL instantiation, UCF constraints


### [`axi-timer`](./axi-timer) — AXI Hardware Timer
Direct register-level programming of the AXI Timer IP core in two modes:

- **Capture mode** — measures clock cycles elapsed between two external hardware events (push button SW5), demonstrating hardware event timestamping
- **PWM mode** — generates a user-configurable PWM signal to control LED brightness, with interactive period and duty cycle input

**Key concepts:** Direct register access via volatile pointers, hardware timers, PWM generation, capture mode, AXI register map


### [`axi-interrupt`](./axi-interrupt) — ARM Interrupt Handling via GIC
Configures the ARM Generic Interrupt Controller (GIC / XScuGic) to handle periodic interrupts generated by the AXI Timer. Binds a user-defined C++ ISR to the timer interrupt line (IRQ_F2P), demonstrating interrupt-driven programming on a Cortex-A9.

**Key concepts:** ARM GIC (XScuGic), ISR registration, interrupt-driven execution, FPGA-to-processor interrupt routing


### [`axi-cdma`](./axi-cdma) — DMA-Accelerated Memory Transfer 
Configures the AXI Central DMA (CDMA) IP core to perform high-throughput memory-to-memory transfers via two AXI high-performance slave ports (HP0 / HP2). An AXI Timer benchmarks the transfer against a software (CPU) loop copy across multiple array sizes.

**Key concepts:** AXI CDMA, DMA vs CPU memory transfer, AXI HP ports, cache coherency (`Xil_DCacheFlush`, `Xil_DCacheInvalidateRange`), performance benchmarking

### Benchmark Results (50 MHz clock, Zynq ZC702)

| Array Size (u32) | SW Cycles | CDMA Cycles | Speedup |
|-----------------|-----------|-------------|---------|
| 1,024           | 1,513     | 1,120       | 1.35×   |
| 4,096           | 6,016     | 2,646       | 2.27×   |
| 8,192           | 12,315    | 4,704       | 2.62×   |
| 16,384          | 24,762    | 8,792       | 2.82×   |
| 32,767          | 49,809    | 16,982      | 2.93×   |
| 1,048,576       | 1,623,301 | 524,888     | 3.09×   |

## Hardware Configuration (XPS)
 
The C++ source files represent only the software layer. Each module required manual hardware system design in Xilinx Platform Studio (XPS) before a single line of code could run — work that is invisible from the code alone but central to how embedded systems on SoCs are built.
 
Across all modules, the following involved:
 
- **IP instantiation** — adding AXI GPIO, AXI Timer, AXI CDMA, and AXI Interconnect IP cores to the Zynq processing system via the XPS IP catalog
- **Bus interface wiring** — manually connecting Master/Slave AXI ports between the processor, interconnects, and peripherals (e.g. `M_AXI_GP1 → axi_interconnect_gp1 → S_AXI_LITE` for CDMA control; `M_AXI → axi_interconnect_hp → S_AXI_HP0/HP2` for DMA data paths)
- **Clock and reset routing** — connecting `FCLK_CLK0` and `FCLK_RESET0_N` signals to every IP instance and BUS_IF port in the system
- **Interrupt routing** — connecting the AXI Timer interrupt output to the processor's `IRQ_F2P` pin for FPGA-to-ARM interrupt delivery
- **High-performance port configuration** — enabling and assigning DDR address ranges to Zynq HP slave ports (HP0 at `0x20000000`, HP2 at `0x30000000`)
- **Address generation** — mapping base addresses for all peripherals, referenced in software via `xparameters.h`
- **UCF pin constraints** — constraining external ports (GPIO outputs, PWM output, capture trigger input) to specific physical pins on the ZC702
- **Synthesis, implementation, and bitstream generation** — compiling the full hardware design through PlanAhead before exporting to SDK

This process reflects how real SoC hardware systems are assembled at the interconnect level — defining the hardware architecture that the software then drives.
